# E-Commerce Website Testing

## Overview

This project demonstrates manual testing activities performed on an E-Commerce web application. The objective was to validate core business functionalities, identify defects, and ensure a smooth user experience across critical user workflows.

The project follows standard Software Quality Assurance (SQA) practices, including test planning, test case design, functional testing, regression testing, API validation, and defect tracking.

## Testing Activities

### Functional Testing

Executed test cases to verify that all application features behaved according to business requirements.

### Smoke Testing

Performed smoke tests after new builds to ensure that critical functionalities were working correctly.

### Regression Testing

Retested previously validated features after defect fixes to confirm that no existing functionality was impacted.

### UI Testing

Validated page layouts, navigation, form validations, responsiveness, and error messages across different user workflows.

### API Testing

Used Postman to verify REST API endpoints, including authentication, product, cart, and order-related services.

### Database Validation

Executed SQL queries to verify backend data consistency for users, products, carts, and orders.
